## 前情提要

---

## 👁️ 核心觀念：機器人的眼睛 (LiDAR)

光達（LiDAR）就像蝙蝠的超音波，會朝四面八方發射雷射並接收反射，藉此測量周遭障礙物的距離。在 ROS 2 中，光達的資料通常有兩種常見的訊息格式：

- `sensor_msgs/msg/LaserScan`（2D）：簡單的平面陣列，依序紀錄每個角度的距離。
- `sensor_msgs/msg/PointCloud2`（3D）：高級的 3D 點雲資料，包含成千上萬個帶有 (X, Y, Z) 空間座標的點。處理起來較複雜，但能看見立體空間。

> 本教學以模擬器中配備 Livox 3D 光達的 OriginBot 為例。其資料頻道為 `/livox/lidar`，格式為 `PointCloud2`。

---

## 🕵️‍♂️ 步驟一：終端機偵探課（排查頻道狀態）

在寫程式接收感測器資料前，先用終端機確認頻道狀態，避開 90% 的「靜默失敗」錯誤。

```bash
# 1. 確認光達是否有在發送資料（檢查心跳）
ros2 topic hz /livox/lidar

# 2. 確認頻道的資料格式（非常重要！）
ros2 topic info /livox/lidar
```

預期結果：會看到頻道的 Type 是 `sensor_msgs/msg/PointCloud2`。這決定 C++ 程式須引入對應標頭檔。

## 💻 步驟二：撰寫 3D 點雲自動煞車節點

### 任務說明

監聽 `/livox/lidar`，解析前方 40 公分寬的走廊內是否有點雲。如果最近的點小於 1.0 公尺，就發布速度 0.0 踩煞車；否則維持速度 0.2 前進。

### 建立程式檔案

進入 `ros2_ws/src/my_cpp_pkg/src/`，建立 `auto_brake.cpp`，內容如下：

```cpp name=auto_brake.cpp
#include "rclcpp/rclcpp.hpp"
#include "geometry_msgs/msg/twist.hpp"
#include "sensor_msgs/msg/point_cloud2.hpp"         // 3D 點雲訊息格式
#include "sensor_msgs/point_cloud2_iterator.hpp"    // ROS 2 內建點雲解析工具

using std::placeholders::_1;

class AutoBrakeNode : public rclcpp::Node
{
public:
    AutoBrakeNode() : Node("auto_brake_node")
    {
        // 1. 建立 Publisher (對馬達發號施令)
        publisher_ = this->create_publisher<geometry_msgs::msg::Twist>("/originbot_1/cmd_vel", 10);

        // 2. 建立 Subscriber (聽取 3D 光達資料)
        // ⚠️ 關鍵知識：rclcpp::SensorDataQoS()
        // 感測器發送資料通常是 "盡力而為 (Best Effort)"。必須加入 SensorDataQoS()，否則會靜默失敗。
        subscriber_ = this->create_subscription<sensor_msgs::msg::PointCloud2>(
            "/livox/lidar", rclcpp::SensorDataQoS(), std::bind(&AutoBrakeNode::cloud_callback, this, _1));
            
        RCLCPP_INFO(this->get_logger(), "3D Auto Brake System Started! Watching for obstacles...");
    }

private:
    void cloud_callback(const sensor_msgs::msg::PointCloud2::SharedPtr msg)
    {
        auto twist_msg = geometry_msgs::msg::Twist();
        float min_forward_distance = 100.0f; // 預設安全距離

        // 讀取點雲 X (前後) 和 Y (左右) 座標
        sensor_msgs::PointCloud2ConstIterator<float> iter_x(*msg, "x");
        sensor_msgs::PointCloud2ConstIterator<float> iter_y(*msg, "y");

        // 走訪每一個 3D 點
        for (; iter_x != iter_x.end(); ++iter_x, ++iter_y) {
            float x = *iter_x;
            float y = *iter_y;

            // 篩選出正前方 40 公分寬走廊的點
            if (x > 0.0 && std::abs(y) < 0.2) {
                if (x < min_forward_distance) {
                    min_forward_distance = x;
                }
            }
        }

        // 小於 1.0 公尺就煞車，否則前進
        if (min_forward_distance > 1.0) {
            twist_msg.linear.x = 0.2; 
            twist_msg.angular.z = 0.0;
            // 節流輸出（每秒只印一次）
            RCLCPP_INFO_THROTTLE(
                this->get_logger(), *this->get_clock(), 1000,
                "Clear ahead (Closest: %.2fm). Moving forward...", min_forward_distance
            );
        } else {
            twist_msg.linear.x = 0.0; 
            twist_msg.angular.z = 0.0;
            RCLCPP_WARN_THROTTLE(
                this->get_logger(), *this->get_clock(), 1000,
                "Obstacle detected at %.2fm! BRAKING!", min_forward_distance
            );
        }

        publisher_->publish(twist_msg);
    }

    rclcpp::Publisher<geometry_msgs::msg::Twist>::SharedPtr publisher_;
    rclcpp::Subscription<sensor_msgs::msg::PointCloud2>::SharedPtr subscriber_;
};

int main(int argc, char * argv[])
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<AutoBrakeNode>());
    rclcpp::shutdown();
    return 0;
}
```

## 🛠️ 步驟三：設定編譯依賴（CMakeLists.txt & package.xml）

### 1. 修改 CMakeLists.txt

在檔案前半段加入 `find_package`，以及新增程式的編譯規則：

```cmake name=CMakeLists.txt
# ... 前面省略 ...
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(geometry_msgs REQUIRED)
find_package(sensor_msgs REQUIRED)  # <--- 新增

# === auto_drive (舊程式) ===
add_executable(auto_drive src/auto_drive.cpp)
ament_target_dependencies(auto_drive rclcpp geometry_msgs)

# === auto_brake (本次新增) ===
add_executable(auto_brake src/auto_brake.cpp)
ament_target_dependencies(auto_brake rclcpp geometry_msgs sensor_msgs)

# 安裝新的 executable
install(TARGETS
  auto_drive
  auto_brake
  DESTINATION lib/${PROJECT_NAME}
)
# ... 後面省略 ...
```

### 2. 修改 package.xml

**注意**：不要刪除 XML 的第一行標頭！  
在 `<buildtool_depend>` 下方加入：

```xml name=package.xml
  <buildtool_depend>ament_cmake</buildtool_depend>
  <depend>rclcpp</depend>
  <depend>geometry_msgs</depend>
  <depend>sensor_msgs</depend>
  <test_depend>ament_lint_auto</test_depend>
```
